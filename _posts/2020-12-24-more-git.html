git: Merging commits from branches

1. Fast-forward merge

This can only be done if there are no other commits to the base branch since the branching took place It is the deafult method that git adopts

git checkout master

git merge featureX

git branch -d featureX

2. Merge commit

git checkout master

git merge featureX

there will be a message here to accept or not the merge B - a no fast-forward merge means that a merge commit will always be created if you want to force a merge commit use the following git merge --no-ff featureX

git branch -d featureX

3. Squash merge

  1. git checkout master
  2. git merge --squash featureX
  3. git commit a)accept or modify the squash message
  4. git branch -D featureX

4. Rebase

Resolving merge conflicts

Essentially any files with conflicts are placed into the working tree after being modified by git - a decision has to involve judgment and the files with modified content are marked up by git with >>>> and <<<< and can be edited, saved, added and them committed.

Again use git log --oneline --graph --all to see the results and check all is good

  1. checkout master
  2. merge featureX
    • a. CONFLICT - both modified fileA.txt
  3. fix fileA.txt
  4. stage fileA.txt
  5. commit the merge commit
  6. delete the featureX branch label

resolving a merge conflict involves 3 commits

  1. the tip of the current branch ( 'ours' or 'mine')
  2. the tip of the branch to be merged ('theirs')
  3. a common ancestor ( 'merge base')

Aborting a merge attempt

git merge --abort
